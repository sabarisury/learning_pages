What Is a Transformer?

The architecture that powers GPT, BERT, T5, and virtually every modern language model.

Before transformers, sequence models used RNNs and LSTMs — processing tokens one by one, left to right. This was slow and struggled to capture long-range dependencies (a word at position 1 affecting position 500).

Transformers threw out recurrence entirely. Instead, every token attends directly to every other token in parallel. Distance doesn't matter — a word can directly reference any other word in the sequence.

The result: faster training, better long-range context, and massively scalable architectures.

The Core Formula
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V
Property RNN/LSTM Transformer
Parallelism ✗ Sequential ✓ Full Parallel
Long-range deps ⚠ Weak ✓ Direct O(1)
Memory usage ✓ O(n) ⚠ O(n²)

Interpretability ✗ Opaque ⚠ Attention maps
Scale ⚠ Limited ✓ Excellent

Tokenization

Converting raw text into a sequence of integer IDs the model can process.

Transformers can't read text directly. We first split text into tokens — the atomic units of meaning — and then map each token to an integer ID from a fixed vocabulary.

✂️

Word-level

Split by spaces. Simple but large vocab. "running" and "run" are separate tokens.

🔤

Subword (BPE)

Most common. "running" → ["run", "##ning"]. Handles unknown words via sub-pieces.

🔡

Character-level

Every character is a token. Tiny vocab but very long sequences.

🧮

Special Tokens

[CLS] start, [SEP] separator, [PAD] padding, [MASK] for masked prediction.

Example: "The cat sat"

Our sentence gets split, special tokens added, then each token gets an ID from the vocabulary lookup table.

Token Embedding

Mapping each integer token ID to a rich, learnable dense vector.

An embedding is a learned lookup table of shape (vocab_size × d_model). When you pass token ID 4937 ("cat"), you get the 4937th row — a vector of 512 numbers that the model has learned captures the semantic meaning of "cat".

These embeddings are trained from scratch along with the rest of the model. Similar words end up with similar vectors (high cosine similarity).

E.shape = (d_model,) = (512,)

Positional Encoding

Injecting position information into embeddings so the model knows word order.

The Problem: Attention is a set operation — it's permutation invariant. If you scramble token order, self-attention produces the same result. We need a way to tell the model "this token is at position 3". That's positional encoding.

The original paper uses sinusoidal positional encodings — deterministic vectors based on position and dimension. Each position gets a unique vector made of alternating sin and cos waves.

Even Dimensions (2i)
PE(pos, 2i) = sin( pos / 10000^(2i/d_model) )
Odd Dimensions (2i+1)
PE(pos, 2i+1) = cos( pos / 10000^(2i/d_model) )
Final Input
X = TokenEmbedding + PositionalEncoding
💡
Why sin/cos? The PE for position pos+k can always be expressed as a linear function of PE at position pos — so the model can learn to attend to relative positions easily.
POSITIONAL ENCODING HEATMAP (pos × dim)
Each row = one position. Each column = one dimension. Blue=negative, Red=positive.
WHY DIFFERENT FREQUENCIES?

Low-index dimensions oscillate slowly (low frequency) — they encode coarse position. High-index dimensions oscillate quickly — they encode fine position. Together they create a unique binary-like code for every position.

📐
Shape unchanged: (1, 5, 512). PE is added element-wise.

Scaled Dot-Product Attention

The core mechanism. Every token asks "how much should I attend to every other token?"

🎯
Intuition: Think of attention as a soft database lookup. You have a Query (what you're looking for), Keys (what each item "advertises"), and Values (what each item actually gives you). Attention computes similarity of Q to every K, softmaxes the scores, and returns a weighted sum of Vs.

Step 1 — Project to Queries, Keys & Values

The input X (shape: seq_len × d_model) is linearly projected three times using learned weight matrices W_Q, W_K, W_V. Each head uses smaller projections of size d_k = d_model/h.

Query (Q)

"What am I looking for?" — Each token produces a query vector.

Q = X @ W_Q
(seq, d_model) @ (d_model, d_k) → (seq, d_k)
Key (K)

"What do I contain?" — Each token produces a key for comparison.

K = X @ W_K
(seq, d_model) @ (d_model, d_k) → (seq, d_k)
Value (V)

"What do I give you?" — The actual content that gets aggregated.

V = X @ W_V
(seq, d_model) @ (d_model, d_v) → (seq, d_v)

Step 2 — Compute Attention Scores

We compute the dot product of every Query with every Key: Q @ Kᵀ. This gives an (seq_len × seq_len) matrix — the raw "similarity score" between every pair of tokens.

We then divide by √d_k. Why? Dot products grow large in high dimensions, pushing softmax into regions of near-zero gradients (vanishing gradients). Dividing by √d_k keeps magnitudes reasonable.

Finally, softmax converts the scores to probabilities that sum to 1 per row. High scores → high attention.

Attention Scores
scores = (Q @ Kᵀ) / √d_k
weights = softmax(scores)
output = weights @ V
ATTENTION WEIGHT MATRIX — hover cells
Row = query token. Column = key token. Value = how much that query attends to that key. Each row sums to 1.0.

Step 3 — Weighted Value Aggregation

Final Output

Multiply the attention weights by the Value matrix. This produces a new representation for each token — a weighted blend of all tokens' values, where the weights reflect how relevant each token was.

Attention(Q,K,V) = softmax(Q·Kᵀ/√d_k) · V

Example: "cat"'s output = 0.62×V(cat) + 0.22×V(sat) + 0.08×V(the) + 0.05×V(on) + 0.03×V(mat)

Multi-Head Attention

Running h=8 attention operations in parallel — each head learns a different "perspective".

A single attention head computes one "relationship map" between all tokens. But language has many layers of relationship: syntactic dependencies, semantic similarity, co-reference, topic continuity...

Multi-Head Attention runs h=8 independent attention heads in parallel, each with its own W_Q, W_K, W_V projections of size d_k = d_model/h = 512/8 = 64.

Each head specializes in different patterns. Then all outputs are concatenated and projected back to d_model.

Multi-Head Formula
MultiHead(Q,K,V) = Concat(H₁,...,H_h) @ W_O
where H_i = Attention(Q@W_Qi, K@W_Ki, V@W_Vi)

Feed-Forward Network

A position-wise MLP that transforms each token independently after attention.

After attention has collected information from all positions, each token's representation is passed through a position-wise FFN independently. The same weights are applied to each position — it's not a function of other positions.

The FFN has two linear layers with a ReLU activation in between. It expands to 4× the model dimension (512 → 2048) then contracts back (2048 → 512).

This expansion-contraction is thought to act as key-value memory — storing factual associations learned during pre-training (see the "Transformer Feed-Forward Layers are Key-Value Memories" paper).

FFN Formula
FFN(x) = max(0, x @ W₁ + b₁) @ W₂ + b₂
where max(0, ·) is the ReLU activation
Shapes
W₁: (512, 2048) → 1,048,576 params
b₁: (2048,)
W₂: (2048, 512) → 1,048,576 params
b₂: (512,)
🧠
Memory hypothesis: The first layer creates "keys" and the second layer creates "values". The ReLU patterns in the middle activate for specific input patterns, retrieving stored factual knowledge.
NETWORK ARCHITECTURE (simplified to fewer nodes)
Input(4) → Hidden with ReLU(8) → Output(4) [simplified; real: 512→2048→512]
RELU ACTIVATION
ReLU(x) = max(0, x) — zeroes out negative activations, introduces non-linearity

Residual Connections & Layer Norm

Two training-stability tricks that make deep transformers possible.

Residual Connection (Skip Connection)

Instead of replacing the input, we add the sublayer output to the original input: x + SubLayer(x). This is a skip connection.

Why? In deep networks (6-12 layers), gradients tend to vanish during backpropagation. The skip connection provides a gradient highway — gradients can flow directly back to earlier layers without passing through many non-linearities.

Layer Normalization

After each residual addition, we normalize across the feature dimension (not the batch dimension like BatchNorm). This stabilizes the distribution of activations during training.

Layer Norm Formula
μ = mean(x), σ² = var(x)
x̂ = (x - μ) / √(σ² + ε)
LN(x) = γ·x̂ + β (γ,β learned)
